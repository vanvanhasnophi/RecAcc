### **第一阶段：问题奠基**

[问题奠基](https://www.notion.so/2f3ae8bbe4898045bd82dd02ddffdd1b?pvs=21)

**目标**：为整个研究建立无可争议的逻辑起点和衡量标准。

- **1.1 数据资产界定**
    - **领域**：智能电商**个性化推荐场景**。
    - **核心资产**：聚焦 **“用户历史行为序列”** （记录级）和 **“商品类目特征”** （特征级）。这两类是驱动推荐效果的核心，且易于模拟和归因。
    - **颗粒度**：明确你的定价对象是[**特征级贡献**](https://www.notion.so/2f3ae8bbe4898045bd82dd02ddffdd1b?pvs=21)，并可聚合至记录级。
- **1.2 终极问题定义**
    - ***将AI模型推流（即推荐/信息流）功能所产生的收益归结为数字资产，并对参与构建该资产的各方（商家、用户）进行衡量***
    - **输入**：一次推荐请求的上下文（用户特征、商品特征、历史行为序列）。
    - **输出**：1) **各输入数据特征的贡献度**；2) **各数据特征的货币价值**。
    - **核心公式定义（必须在此阶段确定）**：
        
        ```
        AI增量收益 (ΔGMV) = 模型预估GMV - 基线GMV
        基线GMV = 全局平均点击率 × 全局平均转化率 × 全局平均客单价
        
        ```
        
- **1.3 核心基线设定**
    - **业务基线**：采用“全局平均法”（如上式）。
    - **归因基线**：为SHAP等算法设定**全零基线**或**均值基线**。

### **第二阶段：数据构建 (Week 2)**

[**数据构建**](https://www.notion.so/2f3ae8bbe489806e9f91e1a57288c983?pvs=21)

**目标**：构建一个内含已知规律、可用于最终验证的“实验场”。

- **2.1 数据来源**：使用**模拟数据**。这是确保研究严谨性的关键，允许你预设因果进行反事实验证。
- **2.2 数据生成设计**
    - 编写数据生成脚本，需包含以下维度：
        - `user_id`, `item_id`
        - **核心特征1**：`user_affinity_score` (用户对商品类目的偏好，0-1)
        - **核心特征2**：`item_popularity` (商品热度，0-1)
        - **噪声特征**：`random_noise_1`, `random_noise_2`
    - **关键**：标签 `click`, `conversion`, `order_value` 的生成逻辑必须**明确且强依赖于** `user_affinity_score` 和 `item_popularity`。例如：
        
        ```python
        p_click = sigmoid(2 * affinity + 1 * popularity + 小随机噪声)
        click ~ Bernoulli(p_click)
        
        ```
        
- **2.3 数据集划分**：按时间划分训练集、验证集、测试集。

### **第三阶段：算法实现 (Week 3-4)**

[算法实现](https://www.notion.so/2f3ae8bbe4898096a8f9d9728f139fc1?pvs=21) 

**目标**：实现从预测到价值分配的核心算法链。

- **3.1 宏观算法框架**
    1. **模型训练**：训练一个简易的DeepFM或MLP模型，预测点击率(CTR)和期望GMV。
    2. **归因计算**：对测试集样本，调用SHAP、IG等计算特征贡献度 `ϕ_i`。
    3. **价值映射**：**这是你自主设计的核心**。
        
        ```python
        # 价值映射函数草案
        def value_mapping(phi_i, delta_gmv, model_perf_auc):
            # phi_i: 特征i的SHAP值（贡献度）
            # delta_gmv: 该次预测的AI增量收益
            # model_perf_auc: 模型AUC，用于折扣模型不完美的影响
            base_value = phi_i * delta_gmv
            discounted_value = base_value * model_perf_auc # 简单举例
            return discounted_value
        
        ```
        
- **3.2 伪代码实现（关键环节）**
    
    ```python
    # 输入：训练好的模型 model，测试样本集 test_data
    # 输出：每个样本中每个特征的价值 valuation_matrix
    
    for each sample in test_data:
        # 1. 计算AI增量收益
        baseline_gmv = calculate_baseline(sample)
        predicted_gmv = model.predict_gmv(sample)
        delta_gmv = predicted_gmv - baseline_gmv
    
        # 2. 计算特征贡献度 (以SHAP为例)
        shap_values = shap_explainer.shap_values(sample)
    
        # 3. 价值映射（调用自主设计的函数）
        for each feature i in sample:
            value_i = value_mapping(shap_values[i], delta_gmv, model.auc_score)
            valuation_matrix[sample_id, feature_i] = value_i
    
    ```
    

### **第四阶段：核心验证 (Week 5-6) 【硬核所在】**

**目标**：用科学实验验证你框架的有效性，这是区别于普通课程设计的核心。

- **4.1 验证实验设计：数据消融**
    - **假设**：你的框架计算出“用户历史行为”特征总价值为 **X** 元。
    - **实验组设计**：在模型中**物理移除**或**噪声化**“用户历史行为”特征，其余一切不变。
    - **对照组**：使用完整特征的模型。
    - **测量**：在相同测试集上，分别运行实验组和对照组，记录**GMV总额的差值**（观测损失 **Y** 元）。
- **4.2 分析与结论**
    - 对比 **X** 与 **Y**。如果 `|X - Y| / Y < 阈值（如20%）`，则验证通过。
    - 使用统计检验（如配对t检验）判断GMV下降是否显著。
    - **无论结果是否符合预期，分析本身都是宝贵发现**。
